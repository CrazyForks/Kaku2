# Kaku Static Website Design

## Visual Theme

Kaku uses a warm command-paper surface: parchment backgrounds, ink-blue actions, restrained rules, and dark terminal/screenshot surfaces. The page should feel closer to Kami and Mole Mac than a generic black-green terminal theme.

## Palette

- `#f5f4ed` parchment page background
- `#fbfaf5` ivory elevated surface
- `#1b365d` ink-blue brand and primary actions
- `#0d7d4d` terminal green for command output only
- `#101013` terminal canvas
- `#d8d4c4` warm line and divider

## Typography

Use system serif stacks for editorial headings and body copy. Use system sans for navigation, controls, and dense labels. Use monospace only for terminal prompts, command snippets, and small technical labels.

## Components

Buttons are pill-shaped, at least 42px tall, and use ink-blue primary or ivory secondary styles. Cards are only used for repeated feature or method items. Terminal windows keep dark chrome and a subtle shadow.

## Layout

The landing page is product-first: Kaku appears as the first viewport signal, followed by a real screenshot, proof points, workflow sections, and final download CTA. Inner content widths are 760px for prose and 1120px for visual grids.

## Verification

Verify with a static server, desktop screenshot, 375px mobile screenshot, and link checking across Chinese and English pages before pushing.
